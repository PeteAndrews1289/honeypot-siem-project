# Security Policy

## Supported scope

Security reports are accepted for the current `main` branch, including the evidence-building and validation scripts, tests, and published sanitized artifacts. Historical commits, deleted branches, forks, and third-party software used by the original lab are outside the supported scope.

This is a completed, decommissioned honeypot case study. There is no live lab infrastructure, service endpoint, account, or credential associated with this repository. Do not probe addresses found in history, attempt to reconnect to former systems, or include live attacker data in a report.

## Reporting a vulnerability

Use GitHub's **Security > Advisories > Report a vulnerability** flow for this repository. Please include:

- the affected file and current commit;
- a concise description of the impact;
- safe, minimal reproduction steps; and
- a suggested remediation, if available.

Do not open a public issue for a suspected vulnerability, secret, raw IP address, credential, or other sensitive data. Use only synthetic or redacted examples, avoid destructive testing, and stop if testing could affect any system or person outside your own environment.

Reports are reviewed on a best-effort basis. This portfolio project does not promise a response or remediation timeframe.
